==========================================================================================

PROPÓSITO DEL DOCUMENTO
-----------------------
Este archivo funciona como:

1. Checklist completo del proyecto.
2. Hoja de ruta por versiones jugables o de prueba.
3. Registro de avance.
4. Documento de traspaso entre Codex y Claude Code.
5. Lista de requisitos de la versión final.
6. Lista de pruebas mínimas antes de fusionar cambios a main.
7. Registro de decisiones, bloqueos, errores y archivos modificados.

REGLA PRINCIPAL
---------------
Cada vez que se cambie de Codex a Claude Code, o de Claude Code a Codex, el nuevo agente debe:

1. Leer este documento completo.
2. Revisar la sección "ESTADO ACTUAL".
3. Revisar la versión en progreso.
4. Leer los últimos commits de Git.
5. Revisar los archivos indicados en "ARCHIVOS MODIFICADOS".
6. No modificar tareas marcadas como completas sin autorización.
7. Actualizar este documento al terminar.
8. Guardar Blender o Unreal.
9. Ejecutar pruebas.
10. Hacer commit antes de cambiar de agente.

SÍMBOLOS DE ESTADO
------------------
[ ] No iniciado.
[~] En progreso.
[X] Completado y probado.
[!] Bloqueado.
[?] Requiere decisión.
[R] Requiere revisión.

IMPORTANTE
----------
Una tarea solo debe marcarse como [X] cuando:

- Está implementada.
- Fue probada manualmente.
- No rompe otras funciones.
- El proyecto abre sin errores.
- Existe un commit asociado.
- Cumple los criterios de aceptación.

==========================================================================================

La prioridad no es construir todo el campus de inmediato.

4. Activos validados.

7. Demo vertical.

12. Optimización.
