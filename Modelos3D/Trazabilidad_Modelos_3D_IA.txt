PROYECTO MEMORIA
INFORME DE TRAZABILIDAD DE MODELOS 3D GENERADOS POR IA
========================================================

PROPÓSITO
---------
Documentar, para cada modelo 3D generado durante esta sesión de trabajo con
Claude (Sonnet 5), el proceso exacto de creación, las instrucciones dadas,
las correcciones solicitadas y las referencias utilizadas.

Este documento NO certifica que ningún activo sea legal, original o esté
listo para publicarse. Esa evaluación requiere una auditoría separada por
parte del equipo del proyecto. Este informe únicamente registra hechos
verificables sobre cómo se generó cada archivo.

Cualquier dato que no se pueda comprobar está marcado explícitamente como
tal, no se completó por inferencia.

MÉTODO DE GENERACIÓN GENERAL (aplica a todos los activos listados)
--------------------------------------------------------------------
Todos los modelos de este informe se generaron mediante scripts de Python
ejecutados en Blender 5.1 en modo headless
(`blender --background --python script.py`), sin abrir la interfaz gráfica
durante la generación. El usuario nunca editó la geometría directamente en
Blender: solo abrió cada archivo ya generado en la interfaz gráfica (modo
normal, sin `--background`) para revisarlo visualmente, y dio
retroalimentación en lenguaje natural en el chat. Cada corrección se aplicó
regenerando el script y volviendo a guardar el mismo archivo .blend.

Todos los materiales son procedurales (nodos Principled BSDF de Blender,
solo con valores de color/rugosidad/metalicidad), creados dentro del mismo
script. Ningún modelo de este informe usa texturas de imagen, fuentes
tipográficas externas, ni assets descargados.

LIMITACIÓN IMPORTANTE: LOS SCRIPTS NO SE CONSERVARON
-------------------------------------------------------
Cada script de Python se escribió como archivo temporal en la raíz de
Modelos-3D (por ejemplo, `build_bookshelf.py`) y se eliminó (`rm`)
inmediatamente después de confirmar que el .blend correspondiente se había
guardado correctamente. No existe copia del código fuente exacto de ningún
script.

La descripción de "lógica del script" en cada entrada de este informe se
reconstruye desde el historial de esta conversación (que Claude sí
conserva como contexto), no desde un archivo verificable de forma
independiente. Si se necesita el código exacto, tendría que volver a
escribirse desde cero; no es una recuperación de un archivo existente.

ACTIVOS EXCLUIDOS DE ESTE INFORME POR INSTRUCCIÓN DEL USUARIO
-----------------------------------------------------------------
Los siguientes elementos fueron eliminados del repositorio y de este
informe a petición explícita del usuario, y no se documentan más allá de
esta nota:

- Un modelo de silla de oficina (ID de registro FUR-LIB-004 / FUR-LIB-006)
  que había sido diseñado usando como referencia visual dos fotografías de
  producto proporcionadas por el usuario (archivos locales en su carpeta
  de Descargas, no incorporados nunca al repositorio). El usuario pidió
  eliminar el modelo y cualquier rastro de esa referencia.
- Dos archivos de murales decorativos y un modelo de estantería con
  libros, preexistentes en el repositorio antes de esta sesión. Estos
  activos fueron generados mediante Claude Code en una sesión anterior,
  según declaración del propietario. No se conservan la sesión ni los
  scripts originales que permitan verificar independientemente su
  proceso de generación. Los activos fueron excluidos del juego por
  decisión del propietario.

-----------------------------------

este informe), el usuario pidió conservar el archivo .py de cada modelo
junto al .blend, en vez de eliminarlo tras confirmar el build. Todos los
activos listados desde ARC-LIB-005 en adelante tienen su script real
