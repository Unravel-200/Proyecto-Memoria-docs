PROYECTO MEMORIA
CHECKLIST MAESTRO PARA EXPORTAR OBJETOS DE BLENDER A UNREAL ENGINE
=================================================================

OBJETIVO
--------
Este documento debe revisarse antes de exportar cualquier objeto creado o modificado en Blender.

Sirve para:

- Static Meshes: muebles, paredes, puertas, herramientas, edificios y objetos.
- Skeletal Meshes: personajes, animales y entidades animadas.
- Objetos interactivos: puertas, cajones, sillas, libros y maquinaria.
- Modelos con animación.
- Modelos con colisiones personalizadas.
- Modelos con LOD.
- Modelos que utilizarán materiales o texturas en Unreal.

SÍMBOLOS
--------
[ ] No revisado.
[X] Correcto.
[N/A] No aplica.
[UE] Pendiente de validación en Unreal.
[R] Requiere corrección.

REGLA DE ESTADOS
----------------
Durante la etapa de creación en Blender, un activo puede avanzar sin haber sido probado todavía en Unreal.

Estados permitidos:

[ ] En modelado.
[ ] Listo en Blender.
[ ] Listo para exportar.
[ ] Exportado.
[UE] Pendiente de validación en Unreal.
[ ] Validado en Unreal.

APROBACIÓN EN BLENDER
---------------------
Un activo puede marcarse como "Listo en Blender" cuando:

[ ] Tiene dimensiones correctas.
[ ] Tiene transformaciones aplicadas.
[ ] Tiene pivote correcto.
[ ] Tiene geometría limpia.
[ ] Tiene normales correctas.
[ ] Tiene UV correcta cuando aplica.
[ ] Tiene materiales organizados.
[ ] Tiene piezas móviles separadas cuando aplica.
[ ] Tiene colisión preparada o planificada cuando aplica.
[ ] Está registrado en Registro_Maestro_Activos_Proyecto_Memoria.txt.

LISTO PARA EXPORTAR
-------------------
Puede marcarse como "Listo para exportar" cuando:

[ ] Pasa todas las revisiones correspondientes de Blender.
[ ] La colección de exportación está limpia.
[ ] La nomenclatura está correcta.
[ ] Se definió el formato de exportación.
[ ] Se guardó el archivo .blend.
[ ] Se registró el commit del repositorio de modelos.

VALIDACIÓN FUTURA EN UNREAL
---------------------------
Las pruebas de importación, primera persona, tercera persona, NavMesh,
colisiones definitivas y rendimiento pueden quedar marcadas como:

[UE] Pendiente de validación en Unreal.

La falta de acceso temporal a Unreal no impide terminar, registrar ni exportar
activos desde Blender.
